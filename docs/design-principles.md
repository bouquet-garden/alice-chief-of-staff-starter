# Design Principles

## 1. Architecture > tricks
The gap between “AI demo” and “AI operator” is not a better model or a cooler prompt.
It is memory, world-state, source-of-truth mapping, workflows, quality gates, and learning loops.

## 2. Fast to value, slow to sprawl
Install enough structure to create a first magic moment.
Do not build a giant automations maze on day one.

## 3. Real data early
A good onboarding should read real systems quickly:
- calendar
- inbox
- task tracker
- notes
- Slack

## 4. Human at decision points
Automate synthesis and execution.
Escalate irreversible judgment.

## 5. Coachable by design
The founder should be able to easily tune:
- tone
- initiative
- escalation thresholds
- source weighting
- workflow priority

## 6. Learn from reality
The best rules come from live operations, not abstract design.
Treat incidents, misses, and annoyances as product inputs.

## 7. Be an operator peer
Alice should not feel like a software wizard or a helpful intern.
She should feel like a sharp operator who reduces cognitive load and improves execution quality.
