# World-State and Accountability

## World-state principle
Alice should reason through canonical entities, not through isolated tool surfaces.

## Core entities
- person
- company
- project
- opportunity / deal
- task
- thread
- meeting
- decision
- idea
- artifact

## Minimum fields per entity
- canonical ID / slug
- aliases
- current state
- owner
- linked systems
- updated_at
- confidence
- open loops

## Cross-surface mapping examples
- email thread -> person + company + project + opportunity
- Slack thread -> task + project + decision
- meeting transcript -> people + company + project + next steps
- Notion task -> project + owner + blocker + stage
- Obsidian note -> concept + project + decision
- calendar event -> meeting + people + project

## What Alice should track operationally
- ball with us / ball with them
- staleness
- escalation thresholds
- relationship tier
- blocker age
- decision dependency
- conflicting signals across systems

## Accountability layers

### Business
- revenue / pipeline
- close rate
- retention / client health
- utilization
- delivery margin

### Operational
- stale follow-ups
- blocker age
- task throughput
- sprint health
- decision load

### Agent / system
- task completion rate
- rework rate
- cost per completed task
- silent stall rate
- escalation rate
- knowledge coverage

### Experiment
Every experiment should have:
- hypothesis
- why now
- metric
- target
- owner
- review date
- keep / kill / modify decision

## Rule
No orphan experiments. No vague "we should remember this" without writing it down.
