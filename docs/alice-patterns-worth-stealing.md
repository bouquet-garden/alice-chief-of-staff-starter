# Alice Patterns Worth Stealing

These are the design structures that make Alice feel like an operator instead of a chatbot.

## 1. 4-layer memory architecture
- identity files
- CONTEXT.md
- daily operational memory
- curated long-term memory

Why it matters: compaction stops being catastrophic.

## 2. Source-of-truth hierarchy
Not every system is equally authoritative.

Default:
- Notion = execution truth
- Email = external commitment truth
- Obsidian = strategic truth
- Slack = team-state signal
- GBrain = world knowledge
- Memory = Alice operating memory

## 3. Ball-with-us / ball-with-them thread ownership
Every important thread should classify:
- who owns the next move
- how stale it is
- when to escalate

This is one of the cleanest “chief of staff, not chatbot” patterns.

## 4. Cross-surface reconciliation
Look for contradictions:
- Slack implies done; Notion says in progress
- email commitment not reflected in task tracker
- meeting decision not written down anywhere

This is management, not automation.

## 5. Incident -> rule
Every meaningful failure should become:
- a documented lesson
- a concrete guardrail
- a reusable operating rule

## 6. Promise -> cron / reminder / check
If the system says “I’ll follow up later,” it should create a durable tracking mechanism.

## 7. Fresh-eyes review loop
For important deliverables:
- create
- review with fresh context
- fix
- review again

## 8. Draft-first overnight work
At night, create drafts, branches, plans, or prepared outputs.
Do not silently make irreversible external commitments.

## 9. Explicit + implicit goals
Track both:
- stated goals
- inferred goals from repeated behavior and attention patterns

## 10. Operator peer + anticipatory chief of staff
Alice should do two jobs at once:
- help with the immediate work
- work ahead toward likely goals, risks, and opportunities

## 11. Magic-moment-first onboarding
Don’t try to install everything before proving value.
Pick one workflow that immediately makes the founder feel understood and relieved.

## 12. Memory as moat
The longer the system runs, the more durable and differentiated it becomes.
The onboarding should make that compounding visible early.
