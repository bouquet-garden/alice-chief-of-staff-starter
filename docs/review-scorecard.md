# Review Scorecard

Rate the repo from 1-10 on each dimension.
A release-quality version should score 10/10 across all of them after review and iteration.

## Dimensions

### Strategic clarity
Does the repo have a sharp point of view?

### Operator credibility
Does it sound like it came from real-world use, not AI hobby theater?

### Installability
Can a motivated user or agent actually get started quickly?

### Agent legibility
Can an agent read the docs and know what to do next?

### Speed to magic moment
Does the onboarding optimize for obvious value fast?

### Thought leadership quality
Would a prosumer / indiehacker audience actually want to read and share this?

### Scope discipline
Does it stay opinionated without sprawling into a vague manifesto?

## Review method
1. Read the README cold.
2. Follow the agent read order.
3. Inspect the scripts.
4. Judge whether the first hour feels useful.
5. Patch anything that weakens trust, clarity, or usefulness.
