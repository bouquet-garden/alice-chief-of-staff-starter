# What Alice Is — and Is Not

This page exists to set expectations clearly.

## Alice is

- a Chief of Staff operating layer for AI agents
- a system for judgment, context, memory, reconciliation, and workflow execution
- a productized onboarding + calibration model
- a durable operating model that gets better from real usage

## Alice is not

- a magic fully autonomous executive replacement
- a promise that every integration is instantly operational on day one
- a guarantee that all workflows are fully automated before configuration
- a generic assistant with a fancy persona

## The right mental model

Alice should feel like hiring a sharp operator who:
- learns your systems
- sets an operating contract
- creates durable context
- reduces drift and cognitive load
- prepares work before taking irreversible action

## The wrong mental model

Alice is not:
- "push button, fully autonomous company"
- "one prompt and now everything works"
- "just another chatbot with nicer docs"

## Why this matters

Clear expectation-setting increases trust, reduces disappointment, and makes the system easier to evolve responsibly.
