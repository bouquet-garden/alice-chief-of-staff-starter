# Onboarding (Confidence-First)

Alice onboarding should create confidence before full automation.

## Goal

By the end of first run, users should feel:
1. Seen
2. Relieved
3. In control
4. Confident this compounds

## Time budget

- 45–90 minutes total for first real setup
- 15–30 minutes to first useful output if integrations are connected

## Flow

### 1) Discovery (10–20 min)
Capture:
- top priorities
- dropped-ball patterns
- real tools in active use
- communication/decision style

### 2) Operating contract (5–10 min)
Set explicitly:
- autonomy ladder
- escalation policy
- quiet hours
- approval gates for external actions

### 3) Source mapping (10–20 min)
Define truth boundaries (example):
- Notion = execution
- Email = external commitments
- Slack = signal
- Notes = strategy
- Transcripts = decision evidence

### 4) First magic moment (15–30 min)
Ship one concrete artifact:
- stale-thread recovery brief
- meeting prep dossier
- top-risk commitments brief
- founder daily operating brief

### 5) Calibration loop (ongoing)
Adjust settings based on corrections and misses.

## Step-by-step onboarding example (concrete dialogue)

### Example founder profile

- Technical operator, team size: 5
- Tools: Telegram, Slack, Gmail, Notion, meeting transcripts, local markdown notes
- Pain: dropped follow-ups, stale threads, unclear ownership

### Step 1 — Discovery (10–20 min)

Example dialogue:

> Alice: "What are the top 3 outcomes this week?"
>
> Founder: "Ship v2 docs, close client follow-up loop, prep Thursday partner meeting."

> Alice: "What usually falls through the cracks?"
>
> Founder: "Email follow-ups and decisions buried in Slack threads."

Output produced:
- `USER.md` baseline profile
- prioritized goals list
- risk assumptions list (where drops happen)

### Step 2 — Operating contract (5–10 min)

Example dialogue:

> Alice: "Autonomy level for week 1?"
>
> Founder: "Draft + internal updates only. No external sends."

> Alice: "Escalate legal/finance immediately, or in daily digest?"
>
> Founder: "Immediate."

Output produced:
- explicit autonomy ladder
- escalation policy
- quiet-hours policy

### Step 3 — Source-of-truth mapping (10–20 min)

Example mapping:
- Notion = execution truth
- Email = external commitment truth
- Slack = live signal
- Obsidian/markdown = strategic truth
- Transcripts = decision context truth

Output produced:
- `SOURCES_OF_TRUTH.md`
- first reconciliation rule set

### Step 4 — First magic moment (15–30 min)

Example selected workflow: **stale-thread recovery brief**

Inputs:
- inbox threads
- Slack references
- current Notion tasks

Output delivered:
- 8 threads needing response
- 3 high-risk commitments with missing owners
- 3 ready-to-send draft responses (still gated for approval)

### Step 5 — Calibration loop (ongoing)

Example update after day 2:
- reduce verbosity from detailed -> strategic
- proactivity from high -> medium
- escalate only P0/P1 outside digest window

Output produced:
- revised settings profile
- quality notes: false positives vs high-value interventions

## What users should expect in the first 24 hours

1. A clear source-of-truth map
2. One high-confidence magic-moment output
3. Safer workflow defaults (draft-first)
4. Explicit behavioral settings
5. A realistic "implemented now vs planned next" view

If these five are missing, onboarding quality is below standard.
